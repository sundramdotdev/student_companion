# Security Policy

## Supported Versions

Currently, only the latest release of Student Companion is supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.1.x   | :white_check_mark: |
| 1.0.x   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within Student Companion, please send an e-mail to the project maintainer via the contact information provided in the in-app About screen or on the GitHub profile. 

We take all security vulnerabilities seriously and will respond to your report as quickly as possible. Please do not report security vulnerabilities through public GitHub issues.

## Offline First Guarantee
As an offline-first application, Student Companion stores all user data locally on the device using Hive. No data is transmitted to external servers without explicit user initiation (e.g., manually sharing an export file). Therefore, network-based vulnerabilities related to user academic data are structurally mitigated by design.
